# Project Knowledge

This is project **state**, not work-item history: what's already decided, what to
reuse, what's already been rejected, and the vocabulary in use. Every role persona
(`.conduit/agents/*.md`) reads the relevant category below before proposing anything
new on a stage-gate card, and writes back here when a card produces something that
should outlive it. Work-item-specific detail stays in `.conduit/work-items/<id>/` — it
never gets promoted here unless it's true beyond that one card.

# Categories

* [Architecture Decisions](decisions/) - accepted/superseded decisions new work must conform to
* [Patterns](patterns/) - established reusable approaches (architecture and UX)
* [Anti-patterns](anti-patterns/) - approaches already tried and rejected, and why
* [Domain Concepts](domain/) - shared vocabulary / glossary
* [Components](components/) - inventory of existing systems/modules
